自我介绍

发表于 2017-02-28 | 分类于 summise

2017年面试自我介绍， 记录此刻的言行， 看着自己的成长与改变

现在处于找工作阶段， 从来没仔细想过面试时， 别人让自我介绍， 应该如何通过短短的几分钟能让别人快速了解自己， 以往面试时仅仅是简单的讲了下自己的工作经历， 通过最近几次面试不是特别顺利， 虽然程序员面试， 着重的是看中技术方面， 但是在自我介绍环节， 给人第一印象多少会有些减分。 觉得 在这方面也应该注意表达方式以及内容， 在这里写个草稿，也留档，在之后可以通过这些文章，来发现自己的成长与变化吧。

自我介绍

您好，我叫王慧英，来自新疆，当然我是汉族人，是典型的西北人性格，属于那种比较开朗向上神经比较大条的，很好相处的人。
我13年毕业于西安交通大学，本科学历，我大学期间曾在新疆大学与西安交通大学各学习了两年，13 年在西安交通大学完成毕业论文答辩。
于14年年底开始从事前端工作至今，我是一个爱折腾的人，使用的硬件工具从 windows 到 mac；从刚开始的使用一些简单的命令行到现在基本上全是通过命令行去工作；使用的技术从刚开始的 html，css，jQuery，到后面的 sass，compass，grunt，gulp，angularJs；使用的编程工具从 editplus，dreamveawer，webstorm，到现在的 vim；
我的第一份工作，是在新疆一个电厂中做电力技术人员，他是一份一旦熟悉熟练就一成不变的工作，我害怕那种一眼就能望到头的工作，所以在工作了一年后，就果断离职了。我很喜欢现在从事的行业前端工作，因为他一直在成长，他的能力一直在扩大，我也是一直在成长，在解除新鲜的技术等。所以我信奉的是生命不止折腾不息，我乐于接受并学习新的知识新的事物。

问答

1. 你为什么会离职

我选择离职，是因为所在的公司都无法为当时的我提供一个工作平台来满足我继续成长的需求，当然主要是指技术方面。
当时也一直处于认识这个行业以及认识自我的一个过程。举个例子，我在第一家公司工作时，已经是前端的负责人了，与各项目负责人沟通，我给手下人员安排工作等等，可是当时他们的开发模式还是传统的模式，前端只负责页面制作，我如果不从第一家公司离开，我不会知道前端已经发展到可以独自负责页面与数据的交互，如react-native可以开发 app，如 node 可以作为后端语言，处理数据等等。
离开上一家公司的也是同样的原因，公司格局太小，

2. 你现在寻求一个什么样的公司

在这两年的时间里，我大概了解了前端的工作，也知道了自己的兴趣点，现在是时候沉淀了，我想找一份工作，是可以长期干下去的，未来几年内可能不会再选择跳槽，那就是最好能有一个轻松愉快的工作氛围，能有一个活跃的技术团队，我不害怕加班，但不喜欢那种为了加班而加班的工作氛围。

3. 你未来的几年规划

特别远的我可能无法计划到，因为计划永远赶不上变化，我近期的计划：
一是工作方面的，技术方面的继续提升，学习新的技术，了解其他人的编程理念等等；
二依旧是自我能力的提升方面，我有报了个英语培训班，希望在我完成培训之后能够真正掌握了英语这门语言，听说读写方面毫无障碍了。虽然自学也是可以的，但是学习语言这一块，我觉得那个语言环境还是很重要的，报班可以去跟更多的人接触沟通交流，而不是闭门造车。
三呢享受生活，活在当下，有什么想做的就去做好了，不要等到以后后悔。

4. 人力面试你有什么问题

技术团队的组成？
进入公司后会负责公司那部分业务，成员有几个？
公司的部门构成？
若是还有什么想知道的，那就是福利待遇了？
允许我自带电脑么？我使用 mac 开发，习惯了他的操作系统，以及他的快速响应，不过也不是必须的。

5. 技术面试你有什么问题

我想了解一下我们技术部门的工作氛围是怎样的，大家属于逗比型的，亦或是属于安静沉默型的，我如果能有幸加入你们团队，我需要调整自己的状态去融入进去；
技术部门的人员组成？
前端使用什么技术？
有没有技术交流，比如分享会什么的？
负责什么业务，开发进度，是筹备中，已经开发一部分了，还是已经成熟，剩下的是版本迭代什么的？
恩，面试官通过咱们短暂的沟通，您能给我一些什么建议么，我想知道哪方面欠缺的以后会着重去修补改正，人只有认识到自己的不足，才知道如何去改正，当然自己对自己的认识毕竟是有局限性的，还需要了解其他人眼中的自己

6. 最近有看什么书么？

一是 css 揭秘，讲的是使用 css 去实现一些常见的效果，可以说是 css 的一个升级，里面涉及到一些 css 不为我们所熟悉的知识点，但知道了之后你会惊讶于，css 太神奇了，还可以这么去实现你想要的效果，算是 css 的一个升级吧
还有一本是 你所不知道的 JavaScript，主要是讲了浏览器在编译解析 js 时是怎么一个流程，恩，算是从实现原理方面来讲解 js，收获也是颇多，让我再读代码时，可以将自己想像成浏览器，更好的更深入的去了解 js 原理吧。
还有一本是非专业书，大冰的乖，摸摸头，大冰是一个行走在路上的作家，主持人等等，这本书主要是记录了他从他的过客那里获得的故事，让我知道了，世界如此丰富多彩，每个人都有他独一无二的故事，有他自己的生活方式等等，我既然没有办法去用脚步丈量这个世界，去了解这个世界，通过书本是最好的一个途径了。我觉得比刷手机刷微博要好的多。

7. 推荐一下你自己

我26岁，已经褪去了年少的青涩，天真无知，但还未沾染市侩。
我入行两年，保持着对本行业的好奇心，以及求知欲，还未成为工作上的老油条（希望我以后也会一直保持我的好奇心与求知欲）。
我性格开朗，属于好相处的人群，不爱抱怨，因为饱受抱怨之苦，不仅给自己带来负面影响，还会传递负能量。
我为人踏实肯干，做一行爱一行，有责任心，应该是自己做好的，那绝对不推辞。
我目前的技术可能不是最好的人选，但如果我能有幸加入，我会以最好的状态迎接新的工作。
请给我一个与您称为同事的机会，相信你的选择不会有错。

8. 介绍一个你认为做的最好的项目

恩，就讲一下我在第一家公司做的一个项目。那是我们公司内部做的一个推广网站，我是全程参与了这个项目，从领导提出想做一个推广网站，我们几个相关同事聚集在一起，商讨网站内容，展现形式， 当时确定是制作响应式的网站。
我就开始去了解学习如何制作响应式网站，也给设计师相应的建议等等，通过这个网站的最终完成，了解了从产品到设计到前端实现的一个流程，也会从用户角度去思考问题，如何设计如何实现，更利于用户使用，而再是简单的不假思索的还原设计稿，前端工程师，也可以说是半个不做图的设计师。
同时也掌握了如何快速掌握一项技术，从确定制作响应式网站时，我通过书本，了解了响应式网站制作的思路，然后又通过研究一个很成熟的响应式框架 Bootstrap 去看他是如何适配不同设备，去看其他的响应式网站如苹果官网，知乎网站等等他们的布局，以及技术上如何实现等等。在前端技术方面，我使用 html5，css3，当时要求兼容至 IE8， 我没有因为 IE8 对 html5，css3 的不支持放弃使用他们，而是寻找了他们的 hack 方法。在技术方面我引入了 css 预处理 scss，compass，如何优雅降级低的去兼容版本浏览器等等。

最近的一个项目，那就是一个销售后台管理平台，这是一个版本更新迭代的项目，这个项目设计得结构很清晰规整，
