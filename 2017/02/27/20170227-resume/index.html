工作经历

壹生源管理平台项目

亿人帮移动官网

开发技术：html5 + css3 + AngularJS + Grunt + Bower + Yeoman
开发工具：Vim + iTerm2 + Chrome
项目链接：http://m.yirenbang.com/dist/index.html
该项目目为一个公益服务平台，集合公益项目展示，参与活动，报名，评论，后续跟踪等一系列流程。在微信公众号内置网站。
我在该项目中负责前端开发工作，本次开发中使用了 AngularJS 以及前端构建工具 Yeoman、Bower、Grunt 进行开发。虽有前面的学习经历，但在实际项目中，还是遇到许多问题，起初代码结构安排不合理，未能将相同功能实现在模板中，作用域及继承未能搞清楚等等问题，为了不影响项目进度，在工作之余一直通过网络以及书本进行学习，对代码进行优化等。 由于该项目之前一直没有项目文档，在接手时花费了很长时间去了解项目代码，编写了项目文档，为后续开发提供便利。

中科软股份有限公司（2014年12月 ~ 2016年3月）

中科软电商推广平台

开发技术：html5 + css3 + Bootstrap + Sass + jQuery + Grunt + Bower + Yeoman
开发工具：WebStorm + Chrome + iTerm2
项目链接：https://xiaoshier.github.io/responsive-web-demo/src/index.html(demo 链接)
该项目是部门的推广网站。这是一个响应式网站，适配 pc 端以及移动端任意尺寸设备。
在这个项目中参与了从产品立项内容以及展现形式的讨论，我提议制作响应式页面一套页面可以适配多端得到了认可。 该项目中使用了 Bootstrap 框架的栅格系统，并借鉴了该框架实现响应式页面的理念，使用了 css3 媒体查询、动画、渐变背景、圆角、选择器等等属性，将一些图片通过 html 和 css实现，减少了图片的加载数量，加快页面的加载速度。针对 IE8 及低版本浏览器采取优雅降级处理，保证使用这些浏览器的用户也能正常访问网站。由于是响应式页面，在开发过程中会去考虑不同分辨率下整体页面比例尺寸如何把控才能有更好的体验效果，在细节上的处理会去与设计师进一步沟通，提出的合理意见并得到了认可。

招商海达微信商城

开发技术：html5 + css3 + MUI + Sass + Zepto
开发工具：Hbuilder + Chrome + Terminal
该项目是招商海达微信商城。
我在这个项目中负责个人中心和商品展示模块儿的页面搭建，使用了 MUI 框架，通过查阅说明文档，能够快速学习并使用了该框架以及该框架配套使用的开发工具。在使用 MUI 开发过程中，由于当时的文档不是十分完善，遇到一些问题如解决自定义滚动条在各设备中样式一致的问题， 只能通过去读源码进行查找问题原因。对我今后的工作很有启发，开发文档的重要性，必须得是一个让所有人都能看懂并能快速接手项目文档。

信诚人寿微理赔项目

开发技术：html5 + css3 + Sass + jQuery
开发工具：WebStorm + Chrome + Terminal
该项目是信诚人寿在微信端推出的理赔项目。
我在此项目中负责前端页面的实现，引用的 css 预编译器 Sass，加快了开发速度；在处理 icon 图标时，使用了compass 合成 css 精灵图，在做真机测试时，发现苹果手机会出现小图标不清晰的问题，通过网络查找到问题原因是苹果手机是 retina 屏具备高像素密度，会导致了图片的模糊，因此使用 css3 媒体查询属性专门针对高 PPI 设备加载大尺寸图片；该网站已有的模块并为对此进行优化，通过如此处理大大的提升了用户体验度。由于该网站以前开发时未形成相关的文档，使得整站没能形成统一风格，发现这个问题后，我提出并尝试着编写了前端开发规范，并编写了通用组件里面包含了如排版、下拉列表、表单的单选多选、按钮、页头页脚等定制化的样式。统一网站整体风格，提升了用户体验度，也为以后开发提供参考基础。通过这个项目的开发知道了 IOS 和 Android 系统在渲染样式方面存在的差异，以及如何在将来的工作中避免再次出现。

中央电视台 — 视听资料TSCU定价系统项目

开发技术：html5 + css3 + jQuery
开发工具：EditPlus + Firefox
PC 端 web 页面开发，这是一个影视资料，图片等定价出售的一个平台。
入行的第一个项目，第一次将理论转为实践，在做这个项目时，遇到了如 IE8 及以下浏览器兼容 css3 新属性如圆角，透明背景，输入框不显示占位符，以兼容模式渲染页面； Chrome 浏览器中输入框有蓝色外边框，字体小于10px 时以 10px 显示等等兼容问题。通过看书以及网上查找解决方案，让我对不同设备以及浏览器间的差异有了一定的认识。后端在套页面时，某些页面在 IE 浏览器中以兼容模式显示布局发生错乱，通过排查发现少了DOCTYPE 声明语句。通过这个问题，不仅知道了 DOCTYPE 的作用，也知道了有必要将开发中的注意事项以文档的形式告知相关同事。通过这个项目学会如何定位问题原因，如何和后端同事沟通配合，也认识到需要去了解学习 W3C 文档，知道每一个元素，每一个属性是如何定义，如何使用。
