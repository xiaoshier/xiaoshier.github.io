git入门

发表于 2016-10-29 | 分类于 git
今天对 git 进行了学习,整理总结记录一下,用来查漏补缺.

昨天在 GitHub Pages 上搭建了自己的 blog, 今天对 git 进行了学习,整理总结记录一下,用来查漏补缺.

设备: MacBook Pro

版本: OS X EI Capitan 10.11.5


1.使用git前准备

由于 mac 预装了 git, 所以免去安装环节.

  • 注册 github 账户

  • 设置SSH Key

    github 上连接已有仓库时的认证,通过使用 SSH 的公开密匙方式认证进行的.也就是说,本地电脑的 git 与远程 github 上的仓库连接是通过 SSH Key 一致时,遍知道你是该用户,我可以上传,下载等(目前的理解如此,不知后面会不会推翻现在的认知)

    打开终端,输入:

    $ ssh-keygen -t rsa -C "your_email@youremail.com"

    输出显示:

    Generating public/private rsa key pair.
Enter file in which to save the key (/Users/xiaoshier/.ssh/id_rsa):

    按回车,不要修改默认路径.

    Enter passphrase (empty for no passphrase):
Enter same passphrase again:

    输入密码,并确认,成功:

    Your identification has been saved in /Users/your_user_directory/.ssh/id_rsa.
Your public key has been saved in /Users/your_user_directory/.ssh/id_rsa.pub. The key fingerprint is: ... ...

    找到.ssh 文件夹,编辑器打开id_rsa.pub文件,复制内容到剪贴板

  • 打开 github 设置ssh,


    添加成功.

2.本地创建 git 项目

  • 本地创建 git 项目

    $ mkdir git-tutorial
$ cd git-tutorial
$ git init

  • 创建文件READEME.md

    添加至本地git仓库, 并保存到本地仓库的历史记录中

    $ git add READEME.md
$ git commit -m "Initial commit"

  • 分支操作

    从 master 分支创建feature-A 分支和 feature-B分支,并合并

    1.创建 feature-A 分支,并处于 feature-A 分支下

    $ git checkout -b feature-A

    2.使用 git branch 查看分支一览表

    $ git branch
* feature-A   //*表示当前所处分支
master

    3.在 feature-A 分支下修改READEME.md 文件,并保历史记录

    $ git add READEME.md
$ git commit -m "add feature-A"

    4.再切回master分支,查看README.md,没有发生变化

    $ git checkbox master

    5.合并分支

    $ git checkout master
$ git merge --no-ff feature-A

    6.查看提交日志(git commit -m “日志内容”)

    $ git log

    7.以图表的形式查看分支

    $ git log -graph

    8.回溯至 feature-A 创建前版本,创建分支feature-B

    $ git reset --hard 9104c361a2e6531fed0ca606a52ade5cbfc0d354

创建分支 feature-B

$ git checkout -b feature-B

    9.在 feature-B 分支下修改READEME.md 文件,并保历史记录

    $ git add README.md
$ git commit -m "branch feature-B"

    10.回复到 feature-A 分支合并后的状态

    $ git reflog

$ git reset --hard 9104c36
$ git merge --no-ff feature-B

    解决冲突后执行,git add , git commit

3.推送至远程仓库github

  • 在 gitbub中创建一个 repositorygit-tutorial,获取HTTPS或 SSH 的值

  • 在终端输入:

    $ git remote add origin https://github.com/username/git-tutorial.git
$ git push -u origin master

    此时在 github 上就可以看见我们的项目了

4. 删除 git 仓库中的文件

$ git rm --cached [filename] || git rm -r --cached [directory]
$ git commit -m 'remove useless files'
$ git push -u origin
